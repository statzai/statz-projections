# statz-projections

FastAPI + Docker service that generates football fixture, team, and player projections for the Statz Laravel app. Runs on a dedicated projection server and is triggered on a schedule (2am UK full run, 1:35pm UK refresh run) by Laravel.

## Deploy workflow

**Never edit files directly on the server** — any change there will be clobbered on the next `git pull`.

1. Edit locally in this clone
2. `git commit` and `git push origin main`
3. SSH to the projection server and pull:

   ```bash
   ssh -p 2223 projections@176.74.18.125
   cd /home/projections/site/statz-projection
   git pull
   docker compose restart statz-projection   # only if the change needs a container restart
   ```

## Data files

`app/data/` contains both source config and runtime-generated data:

- **Committed** — small xlsx config files (`League Weightings.xlsx`, `League Coefficients.xlsx`, `Team Ratings.xlsx`, `Fanteam Mapping.xlsx`, `PL Fantasy Players.xlsx`, `*Promoted Team Ratings.xlsx`)
- **Gitignored** — CSVs, parquet files, generated xlsx (accuracy datasets, model datasets) — all pulled/regenerated by the fetch and projection pipelines

Pre-trained model files live in `app/model-builds/<league>/*.sav` and are gitignored — they're large binaries regenerated by the retrain step.

## Local development

`.env.example` shows the required environment variables. Copy to `.env` and fill in DB credentials to run locally. The real `.env` only exists on the server.
